# MusicFree · PikPak WebDAV 插件

一个把 **PikPak 网盘**作为音乐源接入 [MusicFree](https://github.com/maotoumao/MusicFree) 的插件。通过 PikPak 原生 WebDAV 读取你网盘里的音乐，支持搜索、按文件夹浏览、播放、歌词（自动识别 UTF-8/GBK）、导入歌单，以及一个**会自动反映增删的"实时歌单"**。

> 插件文件：`pikpak-dav.js`　·　歌词解码表：`gbk-index.json`

---

## 功能特性

- 🎵 **搜索整库**：递归扫描配置目录下的全部音频，按歌名/歌手关键字检索（结果缓存，整次会话只扫一次）。
- 📁 **按文件夹浏览**：在"榜单"里以顶层子文件夹（如各歌手）为入口，懒加载，点进去才扫该文件夹。
- ▶️ **直接播放**：返回带鉴权头的地址，播放器自动跟随 PikPak 的 302 跳转到 CDN。
- 📝 **本地歌词**：与歌曲**同名同目录**的 `.lrc` 自动显示，**自动识别 UTF-8 / GBK** 编码。
- 📥 **导入歌单**：把某个目录导入为本地歌单（静态快照）。
- 🔄 **可收藏的实时歌单**：把整库做成一个"全部歌曲"歌单，**收藏后每次打开都实时重查**，PikPak 那边的新增/删除会自动反映（详见下文）。
- 🎯 **文件名解析**：按 `歌手 - 标题` 自动拆分歌手与标题，分隔符可带/不带空格，支持 `-`/`–`/`—`，顺序可配。

---

## 前置条件

1. **PikPak 会员**：WebDAV 是会员功能。
2. **开启 WebDAV**：PikPak App → 设置 → 实验功能 → 开启 WebDAV，会生成一组**专用的 WebDAV 用户名 / 密码**（注意：**不是**你的账号登录密码）。

---

## 安装

在 MusicFree 中「我的 → 插件 → 从网络安装」，填入插件的 **raw 原始地址**：

```
https://raw.githubusercontent.com/cheedonghu/musicfree-plugin/main/pikpak-dav.js
```

或下载 `pikpak-dav.js` 后用「从本地安装」导入。

> **歌词解码表**：插件已内置 `gbk-index.json` 的下载地址（指向本仓库 `main` 分支）。只要 `gbk-index.json` 与插件在同一仓库且可访问，GBK 歌词即可正常解码，无需额外配置。

---

## 配置（用户变量）

安装后点插件「配置」，填写：

| 变量 | 必填 | 说明 |
| --- | --- | --- |
| **用户名** | ✅ | PikPak 实验功能里生成的 WebDAV 用户名 |
| **密码** | ✅ | PikPak 实验功能里生成的 WebDAV 密码 |
| **歌曲根目录** | ➖ | 可留空，默认 `/music`；多个目录用英文逗号分隔。留空且 `/music` 不存在则不处理（不会去扫真正的根目录） |
| **文件名顺序** | ➖ | 填 `歌手-标题`（默认）或 `标题-歌手`，决定文件名里哪段是歌手 |

## 使用说明

### 搜索 / 浏览
- **搜索**：在插件搜索框输入关键字，匹配歌名或歌手。
- **浏览**：进入插件「榜单」，每个配置目录下会列出"…（全部）"入口和各顶层子文件夹，点进去即看该文件夹的歌。

### 歌词
把歌词文件与歌曲**放在同一目录、同名**即可，例如：

```
/music/周杰伦 - 晴天.mp3
/music/周杰伦 - 晴天.lrc     ← 自动加载
```

也兼容 `周杰伦 - 晴天.mp3.lrc` 这种命名。编码 **UTF-8 / GBK 自动识别**（同一目录里两种混用也没问题）。

### 导入歌单（静态快照）
插件「导入歌单」→ 输入要导入的目录路径（留空＝配置的根目录）→ 把该目录下的歌一次性导入为本地歌单。

> ⚠️ 导入的是**快照**：之后 PikPak 那边增删歌曲**不会**自动同步，删除的歌会变成死链，新增的歌需重新导入。要"自动同步"请用下面的实时歌单。

### 实时歌单（推荐 ✅，自动反映增删）
1. 进入插件「推荐歌单」，会看到一个 **「全部歌曲」** 歌单。
2. 打开它 → 点 **❤ 收藏**。
3. 以后从**首页 → 收藏的歌单**打开它，每次都会**实时重新拉取** PikPak 当前内容——新增/删除自动体现，**无需重新导入**。

> 这是**单向实时反映**（PikPak → MusicFree）。真正的"双向同步"无法实现：PikPak WebDAV 是只读的，插件也不能反写网盘或修改本地歌单。

## 注意事项与限制

- **凭据落盘**：MusicFree 会把播放源（含鉴权头）缓存到本地存储，因此 WebDAV 凭据会以明文等价形式留在本机。好在这是**只读、可在 PikPak 实验功能里随时重置**的专用凭据，泄露影响有限。
- **GBK 歌词依赖 `gbk-index.json`**：该表运行时从内置地址下载并缓存（整次会话只下一次）。若你 fork 了本仓库或改了分支/路径，请相应修改 `pikpak-dav.js` 顶部的 `GBK_INDEX_URL`，并确保用 **raw 原始地址**（不是 GitHub 网页的 blob 地址）。
- **深度扫描上限**：递归默认最多 5 层、500 个目录，避免超大目录卡死。
- **`HEAD` 不可用**：PikPak 的 WebDAV 对 `HEAD` 返回 400，插件已避开。

## 本地开发 / 调试

仓库内提供 `debug.mock.js`：用假的 `webdav`/`crypto-js`/`axios` 替换真实依赖，自己编一棵目录树，**不联网、不需要真账号**即可调试扫描/解析/缓存/歌词等逻辑。

```bash
node debug.mock.js
# 打断点：在 pikpak-dav.js 里写 debugger; 然后
node inspect debug.mock.js          # 命令行调试器
node --inspect-brk debug.mock.js    # 配合 Chrome / VS Code
```

改 `debug.mock.js` 顶部的 `MOCK_TREE` / `MOCK_LRC` / `ENV_VARS` 即可模拟不同场景。
